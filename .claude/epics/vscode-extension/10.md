# Task: Create Epic data provider service (read epic.md files)

## Description
Implement a data provider service that discovers and reads Epic files from the `.claude/epics/` directory structure. The service should parse Epic frontmatter and provide a clean API for accessing Epic data.

## Acceptance Criteria
- [ ] Service discovers all Epic directories in `.claude/epics/`
- [ ] Service reads and parses `epic.md` files from each Epic directory
- [ ] Parsed Epic data is returned with proper TypeScript types
- [ ] Service handles missing or malformed Epic files gracefully
- [ ] Service provides methods: `getAllEpics()`, `getEpicById(id)`, `refreshEpics()`

## Technical Details
- Create `services/epicProvider.ts` service
- Use workspace.findFiles() to discover Epic directories
- Read epic.md files using vscode.workspace.fs.readFile()
- Parse frontmatter using the parser from Task 8
- Return Epic objects with frontmatter + content
- Handle errors (file not found, parse errors) with appropriate logging
- Integrate with file watcher (Task 9) for auto-refresh
- Files affected: `src/services/epicProvider.ts` (new file)

## Dependencies
- [ ] Task 8 - Frontmatter parser must be implemented
- [ ] Task 9 - File watcher (optional for auto-refresh feature)
