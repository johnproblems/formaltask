# Task: Add event listeners for notification triggers (4 event types)

## Description
Implement event listeners that trigger notifications based on four key event types within the extension. These listeners will integrate with the notification manager service to display relevant notifications to users at appropriate times.

## Acceptance Criteria
- [ ] Event listener for task status changes implemented
- [ ] Event listener for critical chain updates implemented
- [ ] Event listener for buffer consumption warnings implemented
- [ ] Event listener for project milestone completions implemented
- [ ] All event listeners properly integrated with NotificationManager
- [ ] Event handlers include proper error handling

## Technical Details
- Create event handlers in `src/events/NotificationEventHandlers.ts`
- Use VS Code's event emitter pattern for loose coupling
- Event types to support:
  1. Task status changes (completion, blocking, delays)
  2. Critical chain updates (path changes, conflicts)
  3. Buffer consumption warnings (threshold alerts)
  4. Project milestone completions (progress notifications)
- Register listeners during extension activation
- Ensure proper cleanup on extension deactivation
