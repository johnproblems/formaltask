# Task: Implement TreeDataProvider for Epic/Task hierarchy

## Description
Create a custom TreeDataProvider implementation that reads the epic/task hierarchy from the `.claude/epics/` directory structure. This provider will be the data source for the VS Code tree view, parsing epic.md and task.md files to build the hierarchical structure.

## Acceptance Criteria
- [ ] TreeDataProvider class implemented with getChildren() and getTreeItem() methods
- [ ] Reads epic directories from `.claude/epics/`
- [ ] Parses frontmatter from epic.md and task.md files
- [ ] Correctly builds parent-child relationships (Epic → Tasks)
- [ ] Handles empty directories and missing files gracefully
- [ ] Returns appropriate TreeItem objects for each node

## Technical Details
- Implement `vscode.TreeDataProvider<EpicNode | TaskNode>` interface
- Use existing YAML frontmatter parsing utilities or add gray-matter dependency
- File locations:
  - Create `src/treeview/epicTreeDataProvider.ts`
  - Parser utilities in `src/utils/frontmatterParser.ts`
- Cache parsed data to avoid repeated file reads
- Emit onDidChangeTreeData event for refresh functionality

## Dependencies
- [ ] VS Code Extension API (vscode.TreeDataProvider)
- [ ] File system access (fs/promises)
- [ ] YAML parser for frontmatter (gray-matter or similar)
