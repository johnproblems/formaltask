# Task: Create Task data provider service (read task .md files)

## Description
Implement a data provider service that discovers and reads Task files from Epic directories. The service should parse Task frontmatter, handle task relationships (depends_on, conflicts_with), and provide a comprehensive API for accessing Task data.

## Acceptance Criteria
- [ ] Service discovers all Task markdown files within Epic directories
- [ ] Service reads and parses Task frontmatter from each file
- [ ] Service resolves task dependencies and conflicts
- [ ] Service provides methods: `getTasksByEpic(epicId)`, `getTaskById(id)`, `getAllTasks()`, `refreshTasks()`
- [ ] Service handles missing or malformed Task files gracefully

## Technical Details
- Create `services/taskProvider.ts` service
- Use workspace.findFiles() with pattern `.claude/epics/**/Task*.md`
- Read Task files using vscode.workspace.fs.readFile()
- Parse frontmatter using the parser from Task 8
- Build task dependency graph from depends_on/conflicts_with fields
- Cache task relationships for efficient lookups
- Integrate with file watcher (Task 9) for auto-refresh
- Files affected: `src/services/taskProvider.ts` (new file)

- [ ] Task 9 - File watcher (optional for auto-refresh feature)
