# Task: Implement auto-refresh on progress.md file changes (debounced 500ms)

## Description
Implement automatic refresh of the progress notes panel when the progress.md file is modified. The refresh should be debounced to 500ms to prevent excessive updates during rapid edits, and should preserve the current scroll position when possible.

## Acceptance Criteria
- [ ] FileSystemWatcher monitors progress.md for changes
- [ ] Panel auto-refreshes on file save/modification
- [ ] Refresh debounced to 500ms to batch rapid changes
- [ ] Scroll position preserved after refresh when possible
- [ ] Watcher properly disposed on panel close
- [ ] No refresh triggered by programmatic changes from extension

## Technical Details
- Use vscode.workspace.createFileSystemWatcher() to monitor progress.md
- Implement debounce utility (500ms timeout)
- Store current scroll position before refresh
- Restore scroll position after content update
- Handle case where content length changes significantly
- Add configuration option to enable/disable auto-refresh
- Proper cleanup of watcher on extension deactivation
- Files affected: `src/webview/progressPanelProvider.ts`, `src/utils/debounce.ts` (new)
