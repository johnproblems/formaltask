# Task: Add GitHub label integration (gh api calls + caching)

## Description
Integrate GitHub API to fetch and display label information for tasks that are synced with GitHub issues. Use the `gh` CLI to make API calls and implement caching to minimize API requests and improve performance.

## Acceptance Criteria
- [ ] Fetch GitHub labels using `gh api` for tasks with github field set
- [ ] Display labels in task node (label, tooltip, or decoration)
- [ ] Implement cache with TTL (time-to-live) to reduce API calls
- [ ] Cache invalidation when task is updated
- [ ] Handle API errors gracefully (rate limits, auth issues, network errors)
- [ ] Works only when gh CLI is installed and authenticated

## Technical Details
- Check for `github` field in task frontmatter (e.g., `github: owner/repo#123`)
- Use `gh api` command to fetch issue data:
  ```bash
  gh api repos/{owner}/{repo}/issues/{number}
  ```
- Parse labels from response JSON
- Implement cache in `src/services/githubCache.ts`:
  - Use Map or LRU cache with 5-minute TTL
  - Key: `owner/repo#number`
  - Value: `{ labels: Label[], timestamp: number }`
- Display labels:
  - Option 1: Append to task label `[label1, label2]`
  - Option 2: Show in tooltip
  - Option 3: Use VS Code decorations API
- Error handling:
  - Check if `gh` is installed before making calls
  - Catch API errors and show warning once (don't spam)
  - Fall back to no labels if API unavailable
- File location: `src/services/githubService.ts`
