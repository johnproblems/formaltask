# Task: Setup CI/CD pipeline with coverage gates (90%+ enforcement)

## Description
Implement a robust CI/CD pipeline using GitHub Actions that enforces code quality standards, runs all tests, and ensures minimum 90% code coverage. The pipeline should build, test, and validate the extension on multiple platforms before allowing merges.

## Acceptance Criteria
- [ ] GitHub Actions workflow configured (.github/workflows/)
- [ ] Pipeline runs on push and pull requests
- [ ] Multi-platform testing (Linux, macOS, Windows)
- [ ] 90% minimum coverage enforcement with pipeline failure on non-compliance
- [ ] Build artifacts generated and validated
- [ ] Coverage reports uploaded to CI dashboard

## Technical Details
- Create `.github/workflows/ci.yml` workflow file
- Configure matrix builds for multiple OS (ubuntu-latest, macos-latest, windows-latest)
- Set up Node.js environment with caching
- Run linting, build, and test steps
- Configure nyc to fail if coverage below 90%
- Upload coverage reports (e.g., Codecov or similar)
- Add status badges to README
- Configure branch protection rules recommendation

## Dependencies
- [ ] Task 3 completed (testing infrastructure)
- [ ] GitHub repository with Actions enabled
- [ ] Coverage reporting service account (optional)
