# Task: Implement settings manager service with validation

## Description
Create a centralized settings manager service that handles reading, writing, and validating extension configuration settings. This service will provide type-safe access to settings and ensure configuration values are valid before use.

## Acceptance Criteria
- [ ] SettingsManager class created with singleton pattern
- [ ] Type-safe getters for all 7 configuration settings
- [ ] Validation logic implemented for each setting type
- [ ] Error handling for invalid configuration values
- [ ] Default value fallback mechanism
- [ ] Unit tests achieving 90%+ coverage

## Technical Details
- Create `src/services/SettingsManager.ts`
- Use VS Code's `workspace.getConfiguration()` API
- Implement validation functions for:
  - URL format validation (server endpoint)
  - Numeric range validation (intervals)
  - Enum validation (preferences, themes)
  - Boolean validation
- Provide typed interfaces for settings structure
- Include error logging for validation failures
- Cache frequently accessed settings
- Export singleton instance for use across extension
