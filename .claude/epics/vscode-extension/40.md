# Task: Add menu visibility logic (epic vs task items)

## Description
Implement context-aware menu visibility logic to show different context menu options depending on whether the user right-clicks on an epic item or a task item. This ensures users only see relevant actions for each item type.

## Acceptance Criteria
- [ ] Context menus show different options for epics vs tasks
- [ ] Epic-specific actions are only visible for epic items
- [ ] Task-specific actions are only visible for task items
- [ ] Common actions are visible for both types
- [ ] Menu visibility updates correctly when switching between items
- [ ] No irrelevant menu items appear in any context

## Technical Details
- Use `when` clauses in package.json menu contributions to control visibility
- Define custom context keys to identify item types (e.g., `viewItem == epic` or `viewItem == task`)
- Set `contextValue` property on TreeItem instances in the tree provider
- Example visibility conditions:
  - Epic-only: `viewItem == epic`
  - Task-only: `viewItem == task`
  - Both: `viewItem == epic || viewItem == task`
- Implement logic in tree provider to set appropriate context values
- Consider additional states (e.g., `epic-active`, `task-blocked`) for fine-grained control
- File locations:
  - package.json: `contributes.menus` section with when clauses
  - Tree provider: Set TreeItem.contextValue based on item type
