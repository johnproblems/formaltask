# Task: Implement tree refresh on file system changes (debounced 500ms)

## Description
Set up file system watchers to automatically refresh the tree view when epic or task files are created, modified, or deleted. Implement debouncing to prevent excessive refreshes during bulk operations or rapid consecutive changes.

## Acceptance Criteria
- [ ] FileSystemWatcher monitors `.claude/epics/**/*.md` files
- [ ] Tree view refreshes on file create, change, and delete events
- [ ] Debouncing implemented with 500ms delay
- [ ] Multiple rapid changes trigger only one refresh
- [ ] Watcher is properly disposed when extension deactivates
- [ ] Tree state (expanded/collapsed) preserved after refresh when possible

## Technical Details
- Use `vscode.workspace.createFileSystemWatcher()` API
- Watch pattern: `**/.claude/epics/**/*.md`
- Events to handle:
  - `onDidCreate`: new epic/task added
  - `onDidChange`: epic/task updated
  - `onDidDelete`: epic/task removed
- Implement debounce utility:
  ```typescript
  let refreshTimeout: NodeJS.Timeout | undefined;
  function debouncedRefresh() {
    if (refreshTimeout) clearTimeout(refreshTimeout);
    refreshTimeout = setTimeout(() => {
      treeDataProvider.refresh();
    }, 500);
  }
  ```
- Call `treeDataProvider.refresh()` which fires `onDidChangeTreeData` event
- Store watcher in extension context for proper disposal
- File location: `src/treeview/fileWatcher.ts`
- Integration in `src/extension.ts`
