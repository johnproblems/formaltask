# Task: Configure VSCode extension manifest (package.json extension fields)

## Description
Configure the VSCode-specific fields in package.json that define the extension's behavior, capabilities, commands, configuration options, and activation events. This manifest tells VSCode how to load and interact with the extension.

## Acceptance Criteria
- [ ] Extension activation events defined
- [ ] Contribution points configured (commands, views, menus, keybindings)
- [ ] Configuration schema defined for user settings
- [ ] Extension categories and keywords set
- [ ] Icon and gallery banner configured
- [ ] Main entry point specified correctly
- [ ] Extension manifest validates without errors

## Technical Details
- Configure `activationEvents` array (e.g., onCommand, onView, onLanguage)
- Define `contributes` section:
  - commands: Register extension commands
  - configuration: User-configurable settings with JSON schema
  - views: Custom tree views or webview panels
  - menus: Context menu items and command palette entries
  - keybindings: Default keyboard shortcuts
- Set `main` field to point to bundled extension entry point
- Configure extension metadata:
  - displayName: Human-readable name
  - description: Clear, concise description
  - categories: VSCode marketplace categories
  - keywords: Search keywords
  - icon: Extension icon path
  - publisher: Publisher ID
- Set `engines.vscode` version constraint
- Validate manifest using `vsce`
