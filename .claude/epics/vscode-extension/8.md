# Task: Implement frontmatter parser (YAML parsing with validation)

## Description
Create a robust frontmatter parser that extracts and validates YAML frontmatter from Epic and Task markdown files. The parser should handle errors gracefully and validate against the TypeScript interfaces defined in Task 7.

## Acceptance Criteria
- [ ] Parser extracts YAML frontmatter from markdown files (between `---` delimiters)
- [ ] YAML content is parsed into JavaScript objects
- [ ] Parsed data is validated against EpicFrontmatter/TaskFrontmatter interfaces
- [ ] Validation errors are reported with clear error messages
- [ ] Parser handles malformed YAML gracefully

## Technical Details
- Use `js-yaml` or similar library for YAML parsing
- Create `parsers/frontmatter.ts` file
- Implement `parseFrontmatter<T>(content: string): T` generic function
- Add schema validation using Zod or similar validation library
- Handle edge cases: missing frontmatter, invalid YAML, incomplete fields
- Return parsed frontmatter with proper TypeScript types
- Files affected: `src/parsers/frontmatter.ts` (new file), `package.json` (dependencies)

## Dependencies
- [ ] Task 7 - TypeScript interfaces must be defined first
- [ ] Install js-yaml and @types/js-yaml packages
- [ ] Install validation library (zod recommended)
