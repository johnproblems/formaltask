# Task: Implement notification debouncing logic (5 min per task)

## Description
Add debouncing logic to the notification system to prevent notification spam when multiple events occur in quick succession. Implement a 5-minute debounce period per task to ensure users aren't overwhelmed with repeated notifications for the same task or event.

## Acceptance Criteria
- [ ] Debounce mechanism implemented with 5-minute threshold
- [ ] Tracking system for recent notifications per task
- [ ] Duplicate notification prevention logic working correctly
- [ ] Configurable debounce duration in settings
- [ ] Notification history/cache with automatic cleanup

## Technical Details
- Extend NotificationManager with debouncing capabilities
- Maintain a map of recent notifications with timestamps: `Map<taskId, lastNotificationTime>`
- Implement time-based filtering before showing notifications
- Add cleanup mechanism for stale entries (older than debounce period)
- Key implementation points:
  - Use task ID or unique identifier as debounce key
  - Compare current time with last notification time
  - Only show notification if > 5 minutes elapsed
  - Clear old entries periodically to prevent memory leaks
- Make debounce duration configurable via extension settings
