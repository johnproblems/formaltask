# Task: Implement settings change listeners (hot reload)

## Description
Implement configuration change listeners that enable hot reload of settings without requiring extension restart. This provides a seamless user experience when users modify extension settings through VS Code's settings UI.

## Acceptance Criteria
- [ ] Configuration change event listeners registered
- [ ] Settings automatically reloaded when changed
- [ ] Affected services/components notified of changes
- [ ] Debouncing implemented for rapid changes
- [ ] No extension restart required for setting changes
- [ ] User feedback provided for critical setting changes

## Technical Details
- Use `workspace.onDidChangeConfiguration()` event
- Create event dispatcher in SettingsManager
- Implement observer pattern for settings changes
- Add methods:
  - `onSettingChanged(setting: string, callback: Function)`
  - `notifySettingChange(setting: string, oldValue, newValue)`
- Implement debouncing (300ms) to prevent excessive updates
- Handle specific setting changes:
  - Server URL change: reconnect to new server
  - Sync interval change: restart sync timer
  - Theme change: refresh UI components
  - Debug level change: update logger configuration
- Add setting change notifications for critical changes
- Validate new values before applying
- Rollback on validation failure
