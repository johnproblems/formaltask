# Task: Add status icons (🟢🟡🔴⏭️⚪) based on task status

## Description
Implement visual status indicators for tasks and epics in the tree view using color-coded icons. Map task statuses (open, in_progress, blocked, done, skipped) to appropriate icons that provide quick visual feedback.

## Acceptance Criteria
- [ ] Status icon mapping implemented: 🟢 done, 🟡 in_progress, 🔴 blocked, ⏭️ skipped, ⚪ open
- [ ] Icons display correctly for task nodes
- [ ] Epic nodes show aggregate status icon based on child tasks
- [ ] Icon changes reflect when task status is updated
- [ ] Icons work in both light and dark VS Code themes
- [ ] Fallback to text indicators if icons don't render

## Technical Details
- Update TaskNode and EpicNode classes to set iconPath based on status
- Use VS Code ThemeIcon or custom SVG icons in `resources/icons/`
- Status mapping:
  - `done` → 🟢 (green circle)
  - `in_progress` → 🟡 (yellow circle)
  - `blocked` → 🔴 (red circle)
  - `skipped` → ⏭️ (skip forward)
  - `open` → ⚪ (white circle)
- For epics, calculate aggregate status from children:
  - Any blocked → 🔴
  - Any in_progress → 🟡
  - All done/skipped → 🟢
  - Otherwise → ⚪
- File location: `src/treeview/statusIcons.ts`
