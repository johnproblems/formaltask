# Task: Implement file system watcher for `.claude/epics/` directory

## Description
Create a file system watcher that monitors the `.claude/epics/` directory for changes. This will enable real-time updates in the VSCode extension when Epic or Task files are created, modified, or deleted.

## Acceptance Criteria
- [ ] File watcher monitors `.claude/epics/` directory and all subdirectories
- [ ] Watcher detects file creation, modification, and deletion events
- [ ] Watcher filters for `.md` files only
- [ ] Events are debounced to prevent excessive triggers
- [ ] Watcher properly disposes resources when extension deactivates

## Technical Details
- Use VSCode's `vscode.workspace.createFileSystemWatcher` API
- Create `services/fileWatcher.ts` service
- Implement pattern: `**/.claude/epics/**/*.md`
- Add debouncing (200-500ms) to batch rapid changes
- Emit events that can be consumed by data providers
- Implement proper cleanup in dispose() method
- Files affected: `src/services/fileWatcher.ts` (new file)

- [ ] None - uses VSCode built-in APIs
