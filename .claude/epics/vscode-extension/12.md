# Task: Add caching layer for file reads (5 min TTL, performance optimization)

## Description
Implement a caching layer for Epic and Task data providers to improve performance and reduce file system I/O. The cache should have a 5-minute TTL and automatically invalidate when files change.

## Acceptance Criteria
- [ ] Cache stores parsed Epic and Task data in memory
- [ ] Cache entries have a 5-minute time-to-live (TTL)
- [ ] Cache automatically invalidates when watched files change
- [ ] Cache provides methods: `get(key)`, `set(key, value, ttl)`, `invalidate(key)`, `clear()`
- [ ] Cache reduces redundant file reads and parsing operations

## Technical Details
- Create `services/cache.ts` service
- Implement TTL-based cache with Map data structure
- Store cache entries with: { data, timestamp, ttl }
- Integrate cache with Epic and Task data providers
- Hook into file watcher events to invalidate relevant cache entries
- Add cache statistics for monitoring (hits, misses, evictions)
- Consider using LRU eviction policy for memory management
- Files affected: `src/services/cache.ts` (new file), `src/services/epicProvider.ts`, `src/services/taskProvider.ts`

## Dependencies
- [ ] Task 10 - Epic data provider must exist
- [ ] Task 11 - Task data provider must exist
- [ ] Task 9 - File watcher for cache invalidation
