# Task: Implement action enablement based on item state

## Description
Implement smart enablement logic for context menu actions based on the current state of epic and task items. For example, show "Start" for inactive items and "Stop" for active items, or disable certain actions when they're not applicable to the current item state.

## Acceptance Criteria
- [ ] Menu items are enabled/disabled based on item state
- [ ] "Start" action only appears for inactive items
- [ ] "Stop" action only appears for active items
- [ ] "Mark Blocked" is only available for tasks, not epics
- [ ] Actions that would create invalid states are disabled
- [ ] Menu state updates immediately when item state changes
- [ ] Visual feedback indicates disabled vs enabled actions

## Technical Details
- Extend `contextValue` in TreeItem to include state information:
  - `epic-active`, `epic-inactive`
  - `task-active`, `task-inactive`, `task-blocked`, `task-completed`
- Update `when` clauses in package.json to check for specific states:
  - Start: `viewItem =~ /-(inactive|blocked)/`
  - Stop: `viewItem =~ /-active/`
  - Mark Blocked: `viewItem =~ /^task-/ && viewItem != task-blocked`
- Implement state detection in tree provider:
  - Read task/epic status from files
  - Set appropriate contextValue based on current state
- Ensure tree view refreshes when state changes
- Consider additional constraints:
  - Can't delete active items
  - Can't add dependencies to completed tasks
  - Can't start blocked tasks
- File locations:
  - package.json: Enhanced when clauses
  - Tree provider: Dynamic contextValue generation
  - Command handlers: Validation logic
